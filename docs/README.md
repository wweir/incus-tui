# Documentation Index

本目录集中维护项目设计、架构与实施进度文档，避免文档分散在仓库根目录。

## Documents

- `design.md`：产品设计与模块规划蓝图。
- `implementation-plan.md`：分阶段实施计划与当前进度。

## Update Rules

- 发生架构边界、分层职责、关键数据流变化时，必须同步更新根目录 `ARCHITECTURE.md`。
- 发生里程碑推进（开始/结束阶段）时，必须同步更新 `implementation-plan.md`。
- 发生功能范围变化时，必须同步更新 `README.md` 的“当前实现范围”。
